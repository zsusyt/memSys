# AI 记忆系统 (memSys)

## 项目背景

为 AI 工作流构建一个记忆系统，解决大模型"无状态"的先天限制，让 AI 能够在长期协作中积累上下文、偏好和知识。

## 工作方式

本项目采用**边讨论边进行**的演进式开发方式：

- **讨论即设计**：所有的技术决策、架构调整、需求澄清都通过对话完成，并同步记录到项目文档中。
- **小步快跑**：不追求一次性完成完整设计，而是围绕具体场景逐步迭代。
- **文档先行**：在写代码之前，先把设计思路、接口定义、使用方式写清楚。

## 设计原则

### 语言中立 (Language-Agnostic)

记忆系统的核心接口、存储格式和通信协议必须是跨语言的。任何语言编写的 AI Agent（Python、TypeScript、Go、Rust 等）都应能平等地使用本系统，无需绑定特定运行时或框架。

- **接口层**：优先通过标准协议暴露能力（HTTP REST、gRPC、MCP Server、CLI）
- **存储层**：使用通用格式（JSON、SQLite、Protocol Buffers）
- **实现语言**：可以选择某种语言编写核心服务，但对外暴露的契约必须是语言无关的
- **多语言实现**：未来可能提供多个语言版本的实现，它们必须共享同一套接口契约和存储格式，确保互操作性

### 架构方向

采用**分层记忆模型**：

1. **工作记忆**：当前任务上下文（内存/临时 buffer）
2. **短期记忆**：近期活动与对话摘要（SQLite/JSON）
3. **长期记忆**：持久知识与语义检索（向量数据库 + 嵌入模型）
4. **程序记忆**：规则、模板、规范（Markdown/YAML）

### 技术倾向

- 本地优先，零配置起步
- 存储：SQLite（元数据）+ LanceDB / sqlite-vec（向量检索）
- 接口：先以 Python 库形式实现核心能力，再包装为 MCP Server
- 嵌入模型：支持 OpenAI + 本地模型（Ollama / sentence-transformers）
