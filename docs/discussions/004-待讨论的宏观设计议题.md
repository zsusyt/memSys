## 待讨论的宏观议题

### 1. 记忆的来源与可信度体系
- 用户明确陈述 → 高可信度
- AI 从对话中推断 → 中等可信度
- 从代码/文档中自动提取 → 取决于提取质量
- 系统生成摘要 → 需要验证

**待明确**：不同可信度的记忆在检索时是否应区别对待？低可信度记忆是否应降级或要求二次确认？

### 2. 记忆的检索与注入策略
- 显式检索：AI 主动调用 `memory_search`
- 隐式注入：系统自动检索并塞进 system prompt
- 分层注入：工作记忆直接放 prompt，短期记忆作为参考，长期记忆按需触发

**待明确**：哪种方式更适合代码开发场景？隐式注入是否会导致 prompt 过长或信息噪音？

### 3. 记忆的冲突消解与一致性
- 同一件事在不同时间可能有不同说法（如偏好从 Vue 变为 React）
- 不同来源可能有矛盾观点

**待明确**：系统如何自动发现冲突？发现后是覆盖旧记忆、保留版本历史、还是标记为待澄清？检索时该返回哪一条？

### 4. 记忆的组织与关联
- 标签体系：按标签过滤
- 层级结构：项目 → 模块 → 文件 → 函数
- 知识图谱：记忆作为节点，关系作为边

**待明确**：代码开发场景下，层级结构是否自然但过于复杂？平铺列表是否又太简单？

### 5. 记忆的隐私与隔离
- 项目隔离：项目 A 的记忆不应出现在项目 B
- 用户隔离：多用户共享服务时怎么办
- Agent 隔离：不同 Agent 能否共享记忆

**待明确**：MVP 阶段是否只需考虑"单用户、多项目"隔离？

### 6. 记忆的更新与遗忘策略
- 代码重构后，旧记忆是覆盖、版本化、还是标记过时
- 是否引入时间衰减（旧记忆权重自动降低）
- AI 能否主动"反思"，把零散记忆合并成洞察
- 什么情况下彻底删除，什么情况下软删除/归档

**待明确**：以上策略的具体规则和触发条件。
