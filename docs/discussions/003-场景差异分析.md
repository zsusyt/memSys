# 讨论记录 003：场景差异分析

**日期**：2026-03-29  
**参与者**：用户、Kimi Code CLI

---

## 1. 讨论目标

分析三个候选场景（代码开发助手、个人知识管理、通用 Agent）在记忆内容、检索频率、实时性要求上的差异，为 MVP 场景选择提供依据。

## 2. 场景 A：代码开发助手

### 记忆内容
- 项目架构（模块划分、依赖关系）
- 编码规范（命名约定、风格偏好）
- 历史修改（重构原因、踩过的坑）
- Bug 模式（某类错误常出现在哪）
- 环境配置（数据库连接方式、构建命令）

### 特征
| 维度 | 特征 |
|------|------|
| 检索频率 | **极高**。几乎每次生成代码、审查代码、回答技术问题时都需要检索 |
| 实时性要求 | **高**。文件刚被修改，记忆就应该反映最新状态 |
| 记忆生命周期 | 与项目强绑定。项目废弃后，相关记忆价值骤降 |
| 核心挑战 | 需要与**代码仓库状态同步**。代码变了，记忆也要变，否则会产生"幻觉" |

### 关键洞察
代码场景的记忆不是静态的，而是**代码库的影子**。它最大的风险是**过时**。

---

## 3. 场景 B：个人知识管理

### 记忆内容
- 读书笔记、文章摘要
- 会议纪要、学习心得
- 待办事项
- 人际关系信息（谁负责什么、谁有什么专长）

### 特征
| 维度 | 特征 |
|------|------|
| 检索频率 | **中低**。用户主动提问时才检索 |
| 实时性要求 | **中低**。但"待办事项"类任务记忆需要较高实时性 |
| 记忆生命周期 | **极长**。很多知识是永久有效的，越积累价值越大 |
| 核心挑战 | **信息噪音大**。需要自动筛选、摘要、去重 |

### 关键洞察
这个场景的核心是**知识的可发现性**。用户可能忘了自己存过什么，但能通过提问"挖"出来。

---

## 4. 场景 C：通用 Agent 记忆

### 记忆内容
- 当前任务目标与进度
- 已调用过的工具和结果
- 用户的长期偏好（沟通风格、结果格式）
- 环境状态（浏览器当前页面、文件系统状态）
- 失败教训（某条路走不通）

### 特征
| 维度 | 特征 |
|------|------|
| 检索频率 | **高**。任务执行过程中频繁回顾进度和下一步 |
| 实时性要求 | **极高**。Agent 每执行一步，状态都在变化 |
| 记忆生命周期 | **分化严重**。任务状态是临时的，用户偏好是永久的 |
| 核心挑战 | **工作记忆和长期记忆的频繁切换** |

### 关键洞察
通用 Agent 的记忆系统必须**高度动态**。它更像一个操作系统的内存管理器，而不是静态的知识库。

## 5. 差异对系统设计的影响

### 5.1 对 Schema 的影响

| 场景 | Schema 侧重点 |
|------|---------------|
| 代码开发 | 强关联**文件路径**、**代码片段**、**提交哈希**。记忆应能追溯到具体代码位置 |
| 知识管理 | 需要**来源链接**（URL、书籍名、PDF 页码）、**知识类型**（概念/事实/观点） |
| 通用 Agent | 需要**任务 ID**、**会话 ID**、**工具调用序列**、**状态标记** |

### 5.2 对检索机制的影响

| 场景 | 检索特点 |
|------|----------|
| 代码开发 | **混合检索**：语义搜索 + 精确匹配 + 最近修改优先 |
| 知识管理 | **纯语义检索为主**。对召回率要求高 |
| 通用 Agent | **分层检索**：工作记忆直接放 prompt，短期记忆快速查，长期记忆按需搜 |

### 5.3 对存储引擎的影响

| 场景 | 存储需求 |
|------|----------|
| 代码开发 | 需要与 Git 状态联动，频繁重建索引。向量库要支持**快速增量更新** |
| 知识管理 | 数据量大但更新慢，适合**一次性批量索引** |
| 通用 Agent | 写入极其频繁，需要**低延迟写入**和**高效过期清理** |

### 5.4 对"更新与遗忘"策略的影响

| 场景 | 策略 |
|------|------|
| 代码开发 | **同步覆盖**。代码重构后，旧记忆应被直接替换或标记为过时 |
| 知识管理 | **版本追加**。新观点不覆盖旧观点，由检索时判断相关性 |
| 通用 Agent | **自动过期**。任务状态类记忆设置 TTL，偏好类永久保留 |

---

## 6. 核心能力要求对比

| 场景 | 最考验的能力 |
|------|--------------|
| 代码开发 | **准确性和时效性** |
| 知识管理 | **检索召回率和摘要质量** |
| 通用 Agent | **分层管理和动态调度** |

## 7. MVP 场景推荐

| 场景 | 推荐度 | 理由 |
|------|--------|------|
| **代码开发助手** | ⭐⭐⭐⭐⭐ | 需求最明确，痛点最强，且我们作为开发者最容易验证 |
| **个人知识管理** | ⭐⭐⭐⭐ | 市场大，但竞争者多，差异化需要更多设计 |
| **通用 Agent** | ⭐⭐⭐ | 技术挑战最大，但生态和标准还在演进 |
