# 开源记忆系统参考清单

**整理日期**：2026-03-29  
**说明**：本清单收录与 `memSys` 设计目标相关的开源项目、框架和学术研究，按参考价值分层整理。

---

## 一、综合记忆层/框架（最值得参考）

这些项目提供了一整套记忆的提取、存储、检索、更新机制，与 `memSys` 的目标最为接近。

### 1. Letta（原 MemGPT）

- **GitHub**: https://github.com/letta-ai/letta
- **Stars**: ~19K
- **核心思想**：将 LLM 视为操作系统，上下文窗口作为"虚拟内存"。当主上下文快满时，自动把旧信息换出到外部存储，需要时再换入。
- **值得借鉴**：
  - 主上下文 ↔ 回忆存储 ↔ 归档存储的三级分层架构
  - 通过 Function Call 让 LLM 自主决定何时存取记忆
  - 工作记忆与长期记忆的动态切换机制
- **注意**：2024 年后商业化，但核心框架仍开源。

### 2. Mem0

- **GitHub**: https://github.com/mem0ai/mem0
- **核心思想**：生产级可扩展长期记忆。采用"提取 → 更新"两阶段流水线，自动从对话中抽取记忆并与现有记忆比对合并。
- **值得借鉴**：
  - 记忆的自动提取与整合机制
  - 多用户/多会话隔离（`user_id`、`session_id`、`thread_id`）
  - 图增强版 Mem0g 使用 Neo4j 进行冲突检测和关系建模
- **注意**：2025 年 8 月被 Letta 团队质疑 LOCOMO 基准测试数据存在夸大，但工程实现仍有参考价值。

### 3. LangMem

- **GitHub**: https://github.com/langchain-ai/langmem
- **背景**：LangChain 官方长期记忆 SDK
- **值得借鉴**：
  - 记忆三分类：语义记忆（事实）、情景记忆（经历）、程序记忆（如何做）
  - 背景记忆管理器：自动从对话中提取并整合事实
  - 基于历史数据优化 Prompt 的行为
- **局限**：严重绑定 LangGraph，Python-only，无知识图谱能力。参考价值主要在**设计概念**层面。

### 4. Zep / Graphiti

- **GitHub**: https://github.com/getzep/zep（托管/开源混合）、https://github.com/getzep/graphiti（完全开源）
- **核心思想**：时序知识图谱（Temporal Knowledge Graph）
- **值得借鉴**：
  - 对话历史的图结构化方法
  - 时间感知检索（区分"上周"和"昨天"）
  - 情感分析与深度总结

### 5. Memobase

- **GitHub**: https://github.com/memodb-io/memobase
- **核心思想**：Profile-based 记忆，专注事件演化和人物画像
- **值得借鉴**：
  - 动态更新的"用户画像"维护机制
  - 事件驱动的记忆组织方式

## 二、基础设施层：存储与检索

| 项目 | 类型 | 特点 | 契合度 |
|------|------|------|--------|
| **LanceDB** | 向量数据库 | 文件型、零配置、支持多语言 | ⭐⭐⭐⭐⭐ 本地优先首选 |
| **sqlite-vec** | SQLite 扩展 | 纯 SQLite 内的向量搜索，零依赖 | ⭐⭐⭐⭐⭐ 极简部署 |
| **Chroma** | 向量数据库 | 本地/服务端均可，Python 生态成熟 | ⭐⭐⭐⭐ 多语言支持稍弱 |
| **Qdrant** | 向量数据库 | 高性能，支持过滤查询 | ⭐⭐⭐ 需要独立进程 |
| **FAISS** | 向量检索库 | Meta 出品，纯本地，C++ 核心 | ⭐⭐⭐ 需自行封装 |

## 三、学术研究/早期项目：设计思想参考

这些项目可能不够"生产级"，但概念超前，适合借鉴设计哲学。

### 1. Generative Agents（斯坦福）

- **来源**：模拟人类行为的 Agent 小镇（论文+代码）
- **核心机制**：
  - **记忆流**（Memory Stream）：按时间顺序记录所有事件
  - **反思**（Reflection）：定期总结记忆流，生成高阶洞察
  - **检索**：结合时效性、重要性、相关性三维度打分
- **值得借鉴**：**反思机制**——如何让 AI 主动从零散记忆中提炼更高层知识。

### 2. MemoryBank

- **核心思想**：基于艾宾浩斯遗忘曲线的记忆衰减
- **值得借鉴**：**时间衰减机制**——旧记忆的权重自动降低，模拟人类遗忘。

### 3. A-Mem

- **核心思想**：用 Zettelkasten（卡片盒笔记法）组织记忆
- **值得借鉴**：记忆之间的**双向链接**和**渐进式整理**。

## 四、行业争议与启示

### Letta vs Mem0 基准测试争议（2025 年 8 月）

Letta 团队公开质疑 Mem0 在 LOCOMO 基准测试中的数据夸大。Letta 的实验证明：

> 仅将对话历史存入文本文件，让 GPT-4o-mini 通过 `grep` 和语义搜索查询，在 LOCOMO 上的分数就超过了 Mem0 宣称的 SOTA 结果。

**启示**：
> 记忆系统的性能，可能更多取决于**"Agent 如何管理上下文"**，而不是**"用了多么复杂的检索机制"**。

这对 `memSys` 的设计具有重要指导意义——不应过度工程化检索层，而应把更多精力放在**记忆的注入策略**和**上下文工程**上。
